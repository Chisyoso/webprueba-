PROYECTO TECNOLÓGICO

Nombre del proyecto

timbitoHome – Sistema de automatización doméstica de bajo costo basado en ESP32

⸻

Descripción general del proyecto

timbitoHome es un sistema de automatización del hogar desarrollado con el microcontrolador ESP32, diseñado para permitir el control de dispositivos eléctricos mediante una interfaz web accesible desde teléfonos móviles o computadoras dentro de una red local WiFi.

El sistema funciona mediante una interfaz tipo terminal (timbitoOS), donde el usuario introduce comandos para encender, apagar o consultar el estado de dispositivos conectados. Esta estructura simplifica la interacción y permite un control rápido sin necesidad de aplicaciones complejas.

El proyecto está orientado a demostrar cómo la tecnología IoT puede implementarse de forma accesible, utilizando hardware económico y herramientas de desarrollo web básicas.

⸻

Justificación del proyecto

Necesidad del problema

En la actualidad, los sistemas de domótica comerciales tienen costos elevados, lo que limita su acceso en países en desarrollo. Esto impide que muchas personas puedan implementar soluciones de automatización en sus hogares.

Oportunidad

El crecimiento del uso de teléfonos móviles y redes WiFi domésticas permite desarrollar soluciones locales de bajo costo que sustituyan sistemas comerciales más complejos.

Público objetivo

* Hogares que buscan automatización económica
* Estudiantes de electrónica y programación
* Pequeños negocios que requieren control básico de energía
* Personas interesadas en tecnología IoT

Innovación del proyecto

* Uso de interfaz tipo terminal en lugar de botones tradicionales
* Funcionamiento en red local sin necesidad de internet
* Implementación en hardware económico (ESP32)
* Arquitectura modular que permite expansión futura
* Diseño enfocado en simplicidad y rapidez de uso

⸻

Objetivo general

Desarrollar un sistema de automatización doméstica de bajo costo que permita controlar dispositivos eléctricos mediante una interfaz web conectada a un ESP32 en red local.

⸻

Objetivos específicos

* Diseñar una interfaz web funcional tipo terminal
* Programar el ESP32 para controlar salidas digitales
* Establecer comunicación entre servidor web y microcontrolador
* Implementar comandos básicos de control
* Validar el funcionamiento en red local WiFi

⸻

Fases del proyecto

1. Iniciación

Se identifica la problemática del alto costo de sistemas domóticos y se define la viabilidad del uso del ESP32 como solución principal.

⸻

2. Planificación

* Diseño de arquitectura del sistema (web + ESP32)
* Definición de comandos del sistema
* Selección de componentes electrónicos
* Estructuración del flujo de comunicación

⸻

3. Ejecución

* Desarrollo de la interfaz web (timbitoOS)
* Programación del ESP32 con control de pines
* Pruebas de conexión en red local

* Implementación de respuesta en tiempo real

⸻

4. Cierre

* Evaluación del funcionamiento general
* Documentación técnica del sistema
* Presentación del prototipo
* Análisis de mejoras futuras

Equipo de trabajo

* Desarrollo de interfaz web
* Programación del ESP32
* Diseño de sistema y arquitectura
* Coordinación y pruebas del proyecto

En caso de equipos pequeños, los roles pueden ser combinados según capacidad.

⸻

Recursos necesarios

* ESP32
* Protoboard y cables de conexión
* LEDs, relevadores o sensores básicos
* Computadora para programación
* Teléfono móvil para pruebas
* Red WiFi local

⸻

Arquitectura del sistema

El sistema se divide en dos partes principales:

1. Interfaz web (frontend)
    * Permite la interacción del usuario
    * Envía comandos al sistema
2. ESP32 (backend físico)
    * Recibe comandos
    * Controla dispositivos conectados
    * Responde con estados o confirmaciones

La comunicación se realiza mediante solicitudes HTTP o WebSockets dentro de la red local.

⸻

Decisiones técnicas

* Uso de red local en lugar de internet para mayor velocidad y seguridad
* Interfaz tipo terminal para reducir complejidad visual y mejorar control
* Uso de ESP32 por su bajo costo y capacidad WiFi integrada
* Diseño modular para permitir futuras ampliaciones (sensores, automatización avanzada)

⸻

Riesgos y soluciones

Problemas de conexión WiFi
Se mitiga mediante reconexión automática y validación de estado del dispositivo.

Errores en comandos del usuario
Se soluciona con validación de entradas y mensajes de error claros.

Limitaciones del ESP32
Se optimiza el código para reducir consumo de memoria y mejorar estabilidad.

Falta de experiencia técnica
Se compensa con pruebas constantes y desarrollo iterativo.

⸻

Resultados esperados

* Sistema funcional de control de dispositivos eléctricos
* Interfaz accesible desde dispositivos móviles
* Comunicación estable en red local
* Reducción significativa de costos frente a sistemas comerciales
* Base funcional para mejoras futuras como automatización inteligente

⸻

Lecciones aprendidas

* La planificación es esencial antes de iniciar el desarrollo
* Los sistemas IoT requieren pruebas constantes para estabilidad
* La simplicidad en la interfaz mejora la usabilidad
* El hardware económico puede ser suficiente para soluciones funcionales

⸻

Propuesta de valor

timbitoHome propone una alternativa accesible a la domótica tradicional, enfocada en simplicidad, bajo costo y funcionalidad real.

El sistema permite a usuarios con recursos limitados acceder a tecnología de automatización doméstica sin depender de servicios externos o plataformas pagadas.

⸻

Conclusión

timbitoHome demuestra que es posible desarrollar un sistema de automatización doméstica funcional utilizando hardware económico como el ESP32 y herramientas de desarrollo web básicas.

El proyecto combina tecnología accesible, diseño simple y funcionalidad práctica, ofreciendo una solución viable para la automatización del hogar en contextos de bajo presupuesto.
